# Ekatraa — Fastlane

This folder holds the iOS release tooling. EAS Build still owns the actual
binary compilation (`eas build -p ios --profile production`). Fastlane fills
the gaps EAS does not cover:

| Lane | Purpose |
| --- | --- |
| `fastlane screenshots` | Run the UI-test snapshot suite to regenerate App Store screenshots (6.7", 6.1", iPad). |
| `fastlane metadata` | Push everything under `fastlane/metadata/` and `fastlane/screenshots/` to App Store Connect. |
| `fastlane refresh_store_listing` | Convenience: screenshots + metadata in one go. |
| `fastlane beta` | Upload a pre-built IPA (default `./build/eKatRaa.ipa`) to TestFlight. |
| `fastlane submit_review` | Submit the latest TestFlight build for App Store review. |

## Required env vars (CI / local)

| Variable | Purpose |
| --- | --- |
| `FASTLANE_USER` | Apple ID with App Store Connect access. |
| `FASTLANE_APP_IDENTIFIER` | Defaults to `com.ekatraa.userapp`; override only for staging bundle IDs. |
| `FASTLANE_TEAM_ID` / `FASTLANE_ITC_TEAM_ID` | Developer Portal + App Store Connect team IDs. |
| `FASTLANE_APPLE_APPLICATION_SPECIFIC_PASSWORD` | App-specific password for non-2FA flows. |
| `APP_STORE_CONNECT_API_KEY_PATH` | Preferred — path to a JSON-encoded API key. |
| `APP_REVIEW_EMAIL` / `APP_REVIEW_PHONE` | Override review contact info (defaults live in `Deliverfile`). |
| `APP_REVIEW_DEMO_USER` / `APP_REVIEW_DEMO_PASSWORD` | Demo account passed to Apple review. |

## First-time setup

```bash
bundle config set --local path 'vendor/bundle'
bundle install
bundle exec fastlane snapshot init   # only the first time, to scaffold UI tests
```

Then open `ios/eKatRaa.xcworkspace`, create a UI Testing target named
`eKatRaaUITests`, drop in `SnapshotHelper.swift` (generated by `snapshot init`),
and add `snapshot("01_Home")` style calls where you want shots taken.

## Generating screenshots locally

```bash
bundle exec fastlane screenshots
```

Outputs land under `fastlane/screenshots/<locale>/<device>/`. Commit them so the
metadata lane can pick them up.

## Uploading metadata only

```bash
bundle exec fastlane metadata
```

Binary uploads are intentionally disabled in `Deliverfile`; ship the IPA from
EAS Submit (`eas submit -p ios --latest`) and let Fastlane handle the listing.

## Secrets policy

- Never commit Apple IDs, API keys, or app-specific passwords.
- Use EAS Secrets / GitHub Actions secrets in CI.
- Rotate the App Store Connect API key annually.
